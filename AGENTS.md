# AGENT RULES
- cada funcion nueva debe incluir un bloque de comentario JSDoc encima.
- Todalas las variables y funciones deben ser camelCase
- mas contexto en /.planr

## Additional Coding Standards
- Use TypeScript for all new code in frontend and backend
- Use PascalCase for classes, interfaces, and types
- Use UPPER_CASE for constants
- Use kebab-case for file names
- Import statements should be organized: React imports first, then third-party libraries, then local imports
- Use async/await for asynchronous operations instead of promises
- Error handling should be done with try/catch blocks
- Use meaningful variable and function names
- Avoid magic numbers; use named constants
- Keep functions small and focused on a single responsibility
- Use early returns to reduce nesting
- Validate input parameters in functions
- Use environment variables for configuration
- Follow RESTful API conventions in backend routes
- Use proper HTTP status codes in API responses
- Implement proper authentication and authorization
- Sanitize user inputs to prevent security vulnerabilities
